# Порівняльний аналіз покадрових та послідовних моделей для класифікації відеоданих

## Опис проєкту

Метою роботи є дослідження ефективності двох підходів до класифікації відео: покадрового підходу CNN + Mean Pooling та послідовного підходу CNN-LSTM.

В обох моделях використовується попередньо навчена згорткова нейронна мережа ResNet18 для виділення просторових ознак окремих кадрів відео.

Дослідження проводиться на наборах даних UCF101 та Jester, які містять відеозаписи дій людини та жестів відповідно.

## Використані набори даних

### UCF101

Набір даних для задачі розпізнавання людських дій.

У роботі використовується підмножина класів UCF101. Для кожного відео виконується рівномірний вибір кадрів.

### Jester

Набір даних для класифікації жестів рук.

Для кожного відео виконується або вибір центральної послідовності кадрів, або рівномірний вибір кадрів.

---

## Архітектури моделей

### CNN + Mean Pooling

Кожен кадр відео незалежно обробляється мережею ResNet18.

Після виділення ознак виконується усереднення векторів ознак по часовій осі (Mean Pooling), після чого отриманий вектор подається на класифікатор.

### CNN-LSTM

Кадри відео послідовно обробляються мережею ResNet18 для отримання ознак.

Отримана послідовність векторів ознак подається до мережі LSTM, яка моделює часові залежності між кадрами.

Для класифікації використовується останній прихований стан LSTM.
---

## Попередня обробка даних

Перед подачею до моделі всі кадри проходять етап попередньої обробки:

* зміна розміру зображення;
* обрізання до фіксованого розміру;
* нормалізація за статистиками ImageNet.

Для навчальної вибірки використовуються методи аугментації:

* випадкове обрізання;
* горизонтальне віддзеркалення;
* зміна яскравості, контрастності та насиченості кольорів.

---

## Параметри навчання

У роботі використовуються:

* оптимізатор Adam;
* функція втрат CrossEntropyLoss;
* transfer learning на основі ResNet18;
* механізм Early Stopping.

Основні параметри навчання задаються через файл конфігурації `key.env`.

## Запуск проєкту

### 1. Файл конфігурації

Перед запуском проєкту необхідно створити файл `key.env`, який містить параметри доступу до Kaggle, шляхи до наборів даних та основні гіперпараметри навчання.
Приклад:

```env
KAGGLE_USERNAME=your_username
KAGGLE_KEY=your_key

NUM_FRAMES=16
IMG_SIZE=112
BATCH_SIZE=4
EPOCHS=10
LR=0.0001

UCF101_DATASET_PATH=path/to/UCF101
JESTER_DATASET_PATH=path/to/Jester
```

### 2. Запуск експериментів

Для експериментів з набором даних UCF101:

```bash
bach_try1.ipynb
```

Для експериментів з набором даних Jester:

```bash
jester_dataset.ipynb
```

Клітинки ноутбуків необхідно виконувати послідовно зверху вниз.

## Результати

У процесі експериментів аналізуються:

* точність класифікації (Accuracy);

* графіки точності на навчальній та валідаційній вибірках;

Отримані результати використовуються для порівняння покадрового та послідовного підходів до класифікації відеоданих.
